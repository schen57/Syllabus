Software tools
Basic development tools
Command line interpreter/terminal/console program
We will presume you know how to launch a terminal program, and how to use it to navigate around your file system.
Recommended tools:
Linux users: An xterm running any modern shell should be fine.
Mac OS X users: The Terminal app that comes with OS X is completely adequate for our purposes; it's in the Utilities folder in your Applications folder. More popular among serious developers is iTerm 2, which resembles Terminal at launch, but is more customizable. It's free (donations welcome) from: iTerm2 - Mac OS Terminal Replacement.
Windows users: The Command Prompt program comes with Windows and should be fine for our purposes. A popular alternative, with some extra features like tabs and customization options, is Console (aka Console 2), and open-source project at SourceForge: Console | SourceForge.net.
Text editor/code editor/IDE
You will need a plain text editor (not a word processor) to write software and plain-text documentation files using the Markdown markup language. You could get by with a bare-bones text editor, but consider learning how to use a code editor---a text editor that understands various programming languages, providing capabilities that ease or speed development, such a syntax-aware highlighting, automatic indentation, and code completion.
A very popular cross-platform (Mac/Win/Linux) code editor is Sublime Text 3. This is a commercial app, but version 3 is currently in beta (but very stable) and thus free (and earlier paid versions were 100% functional even without a license). ST3 is a very capable editor by itself, but its real strength comes from the thousands of 3rd-party packages developers have created for it. Its main downside is a somewhat awkward preference/customization system. Expect to struggle with it at first, but if you develop on multiple platforms, or you do both scientific and web programming, it will probably be worth the effort. It's available here: Sublime Text - Download. After installation, the first thing you should do is install the Package Control package manager ( Package Control - the Sublime Text package manager), which simplifies access to the ecosystem of packages. Many ST3 tutorials are available online (the interface is little changed from ST2, so ST2 tutorials remain useful).
An emerging, open-source competitor to ST3 is Atom, both managed and hosted by GitHub.com: Atom
Other popular, free alternatives that will serve our needs well include:
Windows: Notepad++
Mac: TextWrangler
Linux: Text editors that ship with many distributions, such as kate or gedit, provide decent code editing capability.
An integrated development environment (IDE) combines a code editor with other useful development tools, such as built-in command consoles, debugging aids, and variable and class browsers. The Anaconda Python distribution we will be using provides Spyder ( https://code.google.com/p/spyderlib/) a very capable cross-platform IDE. We may provide an introduction to Spyder in a lab.
Many other IDEs are available. If you explore options, keep in mind that different IDEs provide support for different sets of languages. For example, Spyder was initially Python-focused, but has since broadened; however, it doesn't yet support Markdown (but Markdown support is currently being added).
Git
Git DVCS
The course will expose you to the basics of the Git distributed version control system (DVCS, also known as a Source Configuration Management (SCM) system), and use of the GitHub web site for software collaboration and sharing. Some of your assignments will be due as Git repositories ("repos").
The Git web site is: http://git-scm.com/
If you don't already have Git installed on your computer, install it:
Linux users: You likely already have Git (try "which git" at a command line prompt), but if not, you should be able to install it easily via your package manager.
Windows users: Use the installer at the Git web site.
Mac users: A Mac installer is available at the Git web site. However, you should consider installing the Homebrew package manager, and use it to install Git. Homebrew will give you easy access to many packages and programs ported to the Mac from the Linux community and elsewhere. To get Git via Homebrew:
Install Homebrew via the instructions at its home page: Homebrew — The missing package manager for OS X. This involves pasting a 1-line command into an open terminal window (and responding to some prompts).
Again in a terminal window (in any directory), type: "brew install git"

Python
Python environment
We will make heavy use of the Python high-level computing language, specifically the CPython-2.7 implementation (usually just called Python-2.7; the "C" indicates it's implemented in the C language). We will also use many Python packages that are not part of CPython itself (e.g., packages providing fast numerical computing capability, plotting, and of course statistical analysis).
If you already have Python, you may still want to install a new distribution; please read on.
Linux and Mac operating systems come with a version of Python pre-installed, but even on these systems it is highly recommended that one use a separate Python installation for software development. The OS version is often several versions out of date, and will not have all the needed packages. Modifying it (by upgrading Python itself, or by updating or adding packages) can be troublesome since some system software may rely on the OS's Python and package versions.
Windows users will have to install Python (if you have installed a Linux environment for Windows, such as Cygwin or MinGW or a Linux virtual machine, you may have Python, but see the above comment for motivation for installing a separate Python development environment).
You can install a new version of Python, and install and maintain the packages we'll need one by one. I did this for many years. It's doable, but quite a headache, and it makes it hard to share code or collaborate on development, since your environment will differ from others'. We instead strongly urge you to install a specific pre-assembled Python distribution.
Anaconda Scientific Python distribution
We strongly recommend using the Anaconda Python distribution (the Python-2.7 version). It is free, contains most of the software we need, and provides package management and virtual environment capabilities not readily available otherwise. It is also very popular. You'll find it here: Anaconda Scientific Python Distribution.
As instructors, we will assume you are using Anaconda Python-2.7, and we will help with installation and operation issues only for Anaconda.
Anaconda is free and mostly open-source. It is a product of Continuum Analytics, which generates its income via support and via paid-for licenses to some packages that extend Anaconda's numerical capabilities (e.g., optimized libraries, GPU capability). We are not currently planning on covering these additions in class, but as an academic user you can get free access to them.
